okada strict 再学習メモ

現状の問題は、batch_size よりも学習データの混入にあります。strict 版で残っている発話は 2 本の元 MP3 に偏っており、質問文、短すぎる断片、言い淀み混じりの断片がまだ入っています。

判断

- batch_size を上げると必ず精度が落ちるわけではありません。
- ただし今回のようにデータ数が少なく、しかもノイズ混入の疑いがあるケースでは、大きい batch_size は似せる力が落ちやすいです。
- 理由は、1 epoch あたりの更新回数が減り、勾配が平均化されすぎて、少量データの話者特徴に寄り切りにくくなるためです。
- 次回の再学習は batch_size 2 を基準にし、4 を上限にしてください。8 以上は現状では勧めません。

次の進め方

1. [Data/okada_manual_review/20260409-223635_review.tsv](Data/okada_manual_review/20260409-223635_review.tsv) を見て、223635 の断片を 1 本ずつ確認する。
2. [Data/okada_manual_review/20260409-224014_review.tsv](Data/okada_manual_review/20260409-224014_review.tsv) を見て、224014 の断片を 1 本ずつ確認する。
3. initial_action が drop のものは優先的に除外候補として聞き直す。
4. manual_action 列を keep か drop で埋める。
5. manual_action が keep の断片だけで新しい train.list / val.list を作って再学習する。

優先して落とすべき傾向

- 質問文
- 2 秒前後の短すぎる断片
- 「あのー」「そうです」「これ、」など、会話のつなぎだけで始まる断片
- 文末が途中で切れている断片
- 聞いていて相手話者に寄っている断片

補足

- 今回の strict データでは 20260408-225929 など他の元 MP3 は最終的に学習対象に残っていません。
- つまり今のモデルは、話者の多様な話し方を学ぶには量が足りず、しかも 223635 / 224014 の品質に強く依存しています。
- まずは再自動化ではなく、人手でこの 2 本を詰めるのが先です。
